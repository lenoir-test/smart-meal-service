# Тестовое задание для позиции C# Fullstack Developer

Репозиторий содержит выполнение тестового задания для компании ООО "СМАРТ МИЛ СЕРВИС" (Middle Fullstack C# Developer).

## Структура решения

Решение состоит из 5 проектов:
- **SmartMealService.Domain** — общие сущности и интерфейсы.
- **SmartMealService.ApiClient** — реализация клиентов для взаимодействия с сервером (Task 1). Выделен абстрактный интерфейс `ISmsClient`. Реализованы как HTTP-клиент (`SmsHttpClient`), так и gRPC-клиент (`SmsGrpcClient`) в рамках второй части первого задания.
- **SmartMealService.MockServer** — небольшая заглушка на ASP.NET Core (Minimal API + gRPC) для удобства тестирования и отладки клиентов.
- **SmartMealService.ConsoleApp** — консольное приложение (Task 1, Часть 3). Выполняет запросы к серверу, сохраняет данные в БД PostgreSQL через Dapper и взаимодействует с пользователем.
- **SmartMealService.WpfApp** — десктопное WPF-приложение (Task 2). Реализовано по паттерну MVVM (использован `CommunityToolkit.Mvvm`). Позволяет просматривать и изменять переменные среды пользователя.

## Требования
- .NET 8.0 SDK
- PostgreSQL (можно развернуть через Docker, см. ниже)

## Запуск приложения

### 1. База данных
В корне проекта лежит файл `docker-compose.yml`. Для быстрого старта БД выполните:
```bash
docker-compose up -d
```
СУБД будет доступна на локальном порту `54322`. Логин/пароль по умолчанию: `postgres` / `postgres`.
Конфигурация уже прописана в `SmartMealService.ConsoleApp/appsettings.json`. Таблица для блюд создается автоматически при старте приложения (через проверку `IF NOT EXISTS`). В продакшен-коде, разумеется, использовались бы инструменты для миграций (например, EF Core или FluentMigrator).

### 2. Запуск MockServer
Перед запуском консольного приложения необходимо поднять мок-сервер, к которому будут идти gRPC-запросы:
```bash
cd SmartMealService.MockServer
dotnet run
```
Сервер будет слушать порты 5000 (HTTP) и 7004 (HTTPS).

### 3. Запуск Консольного клиента (Задание 1)
```bash
cd SmartMealService.ConsoleApp
dotnet run
```
Логи всех операций пишутся в консоль и дублируются в файл `test-sms-console-app-yyyyMMdd.log` в рабочей директории.

### 4. Запуск WPF-клиента (Задание 2)
Приложение реализует интерфейс в соответствии с макетом (через переопределение стилей DataGrid). Измененные переменные среды сохраняются на уровне пользователя (User scope) и доступны другим приложениям после перезапуска ОС.

Откройте решение в Visual Studio или Rider и запустите проект `SmartMealService.WpfApp`, либо:
```bash
cd SmartMealService.WpfApp
dotnet run
```
Логи будут записываться в файл `test-sms-wpf-app-yyyyMMdd.log`. Список отслеживаемых переменных среды задается в файле `appsettings.json`.
